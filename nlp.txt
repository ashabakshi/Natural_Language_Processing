What is Nlp?
-NLP (Natural Language Processing) is a field of AI (Artificial Intelligence) that helps computers understand,
     interpret, and generate human language — like English, Hindi, etc.

In simple words:
👉 It’s how machines “read, listen, and talk” like humans.

Real World Applications of NLP:

            Chatbots: automated user conversation
            Translation: language convert (EN ↔ HI)
            Sentiment Analysis: detect emotion in text
            Voice Assistants: speech understanding
            Spam Detection: filter unwanted emails
            Autocomplete: predict next words
            Search Engines: understand queries
            Text Summarization: short from long text
            Resume Screening: filter candidates
            Text Generation: AI writing (answers/code)




---

**Approaches of NLP*

* Rule-Based Approach:
  Uses fixed grammar rules and dictionaries
  Limitation: cannot handle new or informal text
  👉 Example: “This movie is lit 🔥” → system may not understand “lit”

---

* Statistical Approach:
  Uses word frequency and probability
  Limitation: ignores full meaning/context
  👉 Example: “bank” (river bank vs money bank) → confusion

---

* Machine Learning Approach:
  Learns from labeled data
  Limitation: depends on training data quality
  👉 Example: if trained only on positive reviews → fails on negative ones

---

* Deep Learning Approach:
  Uses neural networks to understand context
  Limitation: needs large data + high power
  👉 Example: small dataset → model gives wrong predictions

---

* Hybrid Approach:
  Combines rules + ML/DL
  Limitation: complex to build/manage
  👉 Example: more time & effort to maintain both systems

 **NLP Pipeline (ML Perspective Notes)**



**1. Data Collection**

Collect raw text data from different sources

    * Sources: social media, reviews, news, chats , kaggle
    * Type: structured / unstructured(api)
    * Use nlp tools to auto generate data (text augmentation)
    * You can do web scraping to collect data(when data is not available ready-made) 
    * Manual Collection / Crowdsourcing
    👉 ML POV: more data = better model learning

                keep it during preprocessing

### **3. Text Representation (Feature Extraction)**

Convert text → numerical form

    Methods:

    * Bag of Words (BoW) → word count
    * TF-IDF → importance of words
    * Word Embeddings → semantic meaning

    👉 ML POV: model only understands numbers

---

### **4. Dataset Splitting**

    Divide data into:

    * Training set (70–80%)
    * Testing set (20–30%)

    👉 ML POV: prevents overfitting, checks real performance

---

### **5. Model Selection & Training**

    Choose and train model

    * ML models: Naive Bayes, Logistic Regression
    * DL models: RNN, LSTM

    👉 ML POV: model learns patterns from data

    ---

### **6. Evaluation**

    Check model performance

    Metrics:

    * Accuracy
    * Precision
    * Recall
    * F1-score

👉 ML POV: ensures model is reliable

---

### **7. Hyperparameter Tuning**

    Improve model performance
    * Adjust parameters (learning rate, etc.)

👉 ML POV: better accuracy & generalization

---

### **8. Deployment**

Use model in real-world system

* API, web app, chatbot

👉 ML POV: model gives predictions on new data

---

# **🔥 Important Flow (for exams)**

```
Data → Preprocessing → Feature Extraction → Train/Test Split → Model → Evaluation → Deployment
```
